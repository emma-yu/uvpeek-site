你心里可能在想You're probably thinking

老实回答几个问题。 Honest answers to honest doubts.

我已经有 pre-commit hook / husky 了，还需要 Pilo？ I already have pre-commit hooks / husky — do I need Pilo?

它们不冲突，是层叠。 Pre-commit hook 在 commit 时阻断 —— 但你 commit 频繁，尤其和 Cursor / Claude Code 配合时，每几分钟一个 wip commit 是常态。push 才是和外界的真正交接 —— Pilo 在那一刻最后看一眼。 Not competing — they layer. Pre-commit hooks fire at commit time. But with Cursor / Claude Code shipping wip commits every few minutes, the real moment of contact with the outside world is push — that's when Pilo takes its careful look.

而且 hook 只看当前仓库 —— Pilo 看你 Mac 上所有 Git 仓库，跨 repo 一起感知状态。 Also: hooks only see one repo. Pilo watches every Git repo on your Mac, all at once.

macOS only？我是 Linux / Windows 用户。 macOS only? I'm on Linux / Windows.

老实说：是的。Pilo 用了 NSPanel、FSEventStream、UNUserNotificationCenter 这些 macOS 原生 API，才换来菜单栏常驻 + 邮票本浮动 dock + 系统级通知的体验。设计取舍是显式的 —— 我们要写一个像 Mac 软件的 Mac 软件，不是一个跑在 Electron 里的网页。 Honestly — yes. Pilo leans on NSPanel, FSEventStream, UNUserNotificationCenter — the native APIs that make the menu-bar presence, floating stamp dock, and system notifications possible. The design choice is explicit: a Mac app that feels like a Mac app, not a website wrapped in Electron.

代码是 MIT —— 欢迎 fork 一个 cross-platform sibling。 The code is MIT — a cross-platform sibling fork is welcome.

我已经在用 GitGuardian / Gitleaks 了，重复了吗？ I'm already using GitGuardian / Gitleaks — is this redundant?

GitGuardian 是事后通知：你的 key 已经在 GitHub 上、爬虫已经扫到了，邮件才到你手里。Pilo 是事前拦截：还没 push 出去就停下。 GitGuardian is after-the-fact notification: by the time you get the email, the key is already on GitHub and crawlers have it. Pilo is before-the-fact interception — it stops at the push moment, before the key leaves your Mac.

两个工具是层叠互补：Pilo 是第一道，GitGuardian 是兜底。如果 Pilo 漏了什么，GG 还会接住。 They layer: Pilo is the first line, GG is the safety net. If Pilo misses something, GG still catches it.

false positive 误报怎么办？我可不想每次 push 都被打断。 What about false positives? I don't want every push interrupted.

Pilo 有两档误报标记，持久化到本地： Pilo gives you two scopes for false-positive marks, persisted locally:

  • "仅此文件" —— 这个文件这种命中以后都跳过"This file only" — skip this match in this file from now on
  • "仅此规则" —— 这条规则我用不到，全局禁用"This rule" — globally disable a rule you'll never need

另外还有 kill switch：实在烦了一键关闭，24 小时后自动恢复。 There's also a kill switch: one click silences everything, auto-restores after 24 hours.

怎么相信你说的"本地"？凭什么不传我的代码？ How do I trust the "local-only" claim?

最直接的方式 —— 读源码。Pilo 在 github.com/emma-yu/Pilo 完全 MIT 开源。 The directest way — read the source. Pilo is fully MIT-open at github.com/emma-yu/Pilo.

Grep URLSession across the repo — exactly one hit.
